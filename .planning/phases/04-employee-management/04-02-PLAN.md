# Plan 4-02 — Backend API Implementation

Goal: Định nghĩa REST API cho quản lý nhân viên với các trường tên kép.

<tasks>
<task id="4.2.1">
<read_first>
- `backend/src/features/employees/schema.py`
- `backend/src/database.py`
</read_first>
<action>
Định nghĩa Pydantic schemas trong `backend/src/features/employees/schema.py`:
- `EmployeeBase`: Chứa các thông tin cơ bản (ID, Name).
- `EmployeeUpdate`: Thêm `emp_name` (optional), `department`, `shift`, etc.
- `UpdateStatusOut`: Trạng thái của quá trình Update Registry (thay thế cho Rebuild).
</action>
<acceptance_criteria>
- Schema `EmployeeOut` hiển thị `emp_name` từ DB/Excel.
- `EmployeeUpdate` cho phép cập nhật metadata mà user cung cấp.
</acceptance_criteria>
</task>

<task id="4.2.2">
<read_first>
- `backend/src/features/employees/router.py`
- `backend/src/features/employees/service.py`
</read_first>
<action>
Implement endpoints trong `backend/src/features/employees/router.py`:
- `GET /api/employees`: List employees với pagination và filters.
- `POST /api/employees/update-registry`: Trigger background task update registry (đã đổi tên từ rebuild).
- `GET /api/employees/update-status`: Poll status của update process.
- `DELETE /api/employees/{id}`: Trigger hardware-only delete. Trả về kết quả check máy online/offline.
- `PUT /api/employees/{id}`: Cập nhật thông tin nhân viên (trigger `update_employee_info`).
- `GET /api/employees/{id}/biometric-coverage`: Get coverage status.
</action>
<acceptance_criteria>
- Endpoints trả về thông tin chi tiết về các máy không kết nối được khi xóa/rename.
- Endpoint `PUT` hoạt động và đồng bộ tên xuống máy chấm công.
</acceptance_criteria>
</task>
</tasks>
