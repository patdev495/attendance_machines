# Plan 4-02 — Backend API Implementation

Goal: Định nghĩa REST API cho quản lý nhân viên.

<tasks>
<task id="4.2.1">
<read_first>
- `backend/src/features/employees/schema.py`
- `backend/src/database.py`
</read_first>
<action>
Định nghĩa Pydantic schemas trong `backend/src/features/employees/schema.py`:
- `EmployeeBase`, `EmployeeCreate`, `EmployeeUpdate`: Fields tương ứng với table `EmployeeLocalRegistry`.
- `EmployeeOut`: Bao gồm các fields từ registry và optional biometric status.
- `DeleteStatusOut`: Trạng thái xóa nhân viên.
- `BiometricCoverageOut`: Kết quả coverage per machine.
</action>
<acceptance_criteria>
- Schema `EmployeeOut` có trường `source_status`.
- Schemas tuân thủ định dạng Pydantic v2 (nếu đang dùng) hoặc v1 tùy theo project config.
</acceptance_criteria>
</task>

<task id="4.2.2">
<read_first>
- `backend/src/features/employees/router.py`
- `backend/src/features/employees/service.py`
</read_first>
<action>
Implement endpoints trong `backend/src/features/employees/router.py`:
- `GET /api/employees`: List employees với pagination và filters (source_status, search).
- `POST /api/employees/rebuild-registry`: Trigger background task rebuild registry.
- `GET /api/employees/rebuild-status`: Poll status của sync (reuse logic sync_status).
- `DELETE /api/employees/{id}`: Trigger background delete from all machines.
- `GET /api/employees/{id}/delete-status`: Poll delete status.
- `PUT /api/employees/{id}/name`: Update employee name.
- `GET /api/employees/{id}/biometric-coverage`: Get coverage status.
</action>
<acceptance_criteria>
- Tất cả endpoints đều có prefix `/api/employees`.
- Endpoints background tasks trả về 202 Accepted.
- Sử dụng `get_db` dependency.
</acceptance_criteria>
</task>
</tasks>
