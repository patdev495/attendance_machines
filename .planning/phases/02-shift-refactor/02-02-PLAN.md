# Wave 2 Plan: Automated Testing Suite

This wave implements a robust testing infrastructure to guarantee the accuracy of attendance statistics and shift rule calculations.

## 🎯 Objectives
- **Verification**: Ensure that `compute_day_stats` handles early/late, OT, and midnight shifts correctly.
- **Regression Proof**: Protect the core business logic as we proceed to multi-language and UI refactors.

## 📝 Tasks

### 1. Test Infrastructure Setup
- [ ] **Create** `backend/tests/` directory.
- [ ] **Create** `backend/tests/conftest.py`:
    - Configure an in-memory SQLite database for testing.
    - Setup fixtures for `ShiftRule` and `EmployeeMetadata` mock data.
- [ ] **Add** `pytest` to `requirements.txt` (if missing).

### 2. Unit Testing Shift Utility
- [ ] **Create** `backend/tests/test_shift_utils.py`:
    - Test default rules (Night/Day).
    - Test "Xưởng 1" override rules.
    - Test fallback logic for unknown departments.

### 3. Unit Testing Stats Utility
- [ ] **Create** `backend/tests/test_stats_utils.py`:
    - **Normal Day Shift**: 08:30 → 17:30 (Late 30m, Early 0m, 8h std, 0.5h OT).
    - **Midnight Shift**: 20:05 → 05:10 (Late 5m, Early 0m, 8h std, 0h OT).
    - **Xưởng 1 Day**: 07:55 → 20:10 (12h standard, 0h OT).
    - **Missing Tap**: 08:00 → None (Verify it returns 0h, 0 OT, and note placeholder).
    - **Early Leave**: 08:00 → 16:50 (Verify 10m early leave).

## 🧪 Verification Plan

- [ ] **Execution**: Run `pytest backend/tests/` and confirm all tests pass.
- [ ] **Coverage Check**: Verify that `shift_utils.py` and `stats_utils.py` are fully exercised by the new tests.
